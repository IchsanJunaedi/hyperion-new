# Ichsan — Working Style

## Communication
- Bahasa Indonesia setiap saat — jangan campur bahasa tanpa alasan teknis
- Jawaban singkat, langsung ke inti — tidak perlu panjang lebar
- Tidak suka AI yang over-explain atau justify terlalu banyak

## How He Gives Corrections
- Koreksi biasanya pendek dan langsung: "gausah", "engga", "hapus aja"
- Tidak butuh acknowledgment panjang — langsung implement saja
- Kalau dia bilang simplify scope: jangan tanya alasan, langsung ikuti

## Planning Preference
- Untuk fitur besar: **minta plan dulu**, baru implement setelah dia setuju
- Format plan yang dia suka: ringkas, ada tabel file yang diubah, ada pertanyaan klarifikasi di akhir
- Kalau dia revisi scope di tengah plan: langsung adjust, jangan defend rencana awal

## Design Taste
- Premium, dark, glassmorphism — tidak suka tampilan plain/basic
- Jika hasil kerja terlihat seperti minimum viable product: itu gagal
- Selalu perhatikan screenshot yang dia kirim — ada detail visual yang perlu diperhatikan

## After Every Task
- Dia expect: commit + push sudah selesai tanpa diminta
- Commit per kategori — bukan satu commit untuk semua perubahan
